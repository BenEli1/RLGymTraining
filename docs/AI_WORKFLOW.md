# AI Workflow

AI assistance was used for repository scaffolding, implementation planning, documentation drafting, and test generation. Human review is still required for correctness, assignment fit, and any real dataset mapping.

Prompt categories:

- audit current repository and assignment requirements
- design source package and config layout
- implement LSTM world model, REINFORCE, A2C, reward, and tests
- write honest docs and experiment TODOs
- run quality checks and fix failures

Manual review checklist: verify course terminology, verify dataset columns, inspect reward formula, inspect safety thresholds, run commands on the final machine, and confirm no generated text claims unrun results.

Limitations: AI can produce plausible but wrong equations or overclaim experiment outcomes. This repo documents unrun experiments explicitly as planned.
